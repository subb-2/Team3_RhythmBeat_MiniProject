# 리듬 게임 악곡 데이터 포맷 가이드 (새로운 박자 간격 기반)

## 1. 데이터 작성 형식
각 줄은 띄어쓰기로 구분된 두 개의 값으로 구성됩니다.
포맷: `[레인(Lane Hex)] [다음 노트까지의 박자(Gap Beats)]`

- 첫 번째 값: 노트가 떨어질 **레인의 16진수 값(Hex)**.
- 두 번째 값: 이 노트가 떨어지고 난 후, **다음 노트가 나올 때까지 대기할 박자(Beat)**. (예: 4분음표=1.0, 8분음표=0.5, 16분음표=0.25)

---

## 2. 레인별 계이름(음높이) 배치 규칙
보컬 멜로디(또는 주선율) 악보를 4개의 레인으로 옮길 때 사용하는 기준입니다.
총 8개의 기본 계명(도~높은 도)을 4개의 레인에 2개씩 균등 분배합니다.

* **레인 1 (16진수: `8`)** - [A키, 붉은색]
  - **계이름 조건**: **도(C), 레(D)** 및 그 이하의 가장 낮은 음
  - 느낌: 곡의 가장 바닥을 받쳐주는 저음

* **레인 2 (16진수: `4`)** - [S키, 주황색]
  - **계이름 조건**: **미(E), 파(F)** 
  - 느낌: 저음에서 중음으로 넘어가는 소리

* **레인 3 (16진수: `2`)** - [D키, 노란색]
  - **계이름 조건**: **솔(G), 라(A)** 
  - 느낌: 중음에서 고음으로 넘어가는 소리

* **레인 4 (16진수: `1`)** - [F키, 초록색]
  - **계이름 조건**: **시(B), 높은 도(High C)** 및 그 이상의 가장 높은 음
  - 느낌: 곡의 클라이맥스를 찌르는 고음

> **동시치기(화음) 계산법**
> 만약 멜로디에 화음이 있어서 '미(4)'와 '솔(2)'이 동시에 나온다면: 
> 4 + 2 = 6 이므로 레인 값에 `6`을 적습니다.

## 3. 롬 데이터(mem) 작성 예시
우리 파이썬 코드는 적힌 `박자`와 `BPM`을 조합해 자동으로 완벽한 '초(Second)' 단위로 변환해 줍니다.

```text
0 32.0   // [시작 대기] 0번 레인(투명 노트). 노래 시작 전 전주 32박자(8마디) 대기
8 0.5    // [도] 레인 1에 노트 생성 후, 8분음표(0.5박자) 대기
4 0.25   // [미] 레인 2에 노트 생성 후, 16분음표(0.25박자) 대기
2 0.25   // [솔] 레인 3에 노트 생성 후, 16분음표(0.25박자) 대기
1 1.0    // [높은 도] 레인 4에 노트 생성 후, 4분음표(1.0박자) 대기
```
